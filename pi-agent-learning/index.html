// what-is-pi

Pi 是什么？

Pi 是一个极简主义的终端 Coding Agent —— 不预设工作流，而是让你通过扩展来塑造它。

✦ 终端原生 · TUI 界面
✦ 4 个内置工具：read / bash / edit / write
✦ TypeScript 扩展体系
✦ 树形会话 & 分支
✦ SDK 可嵌入

设计哲学

❌ 没有 MCP — 用 CLI 工具 + README 即可
❌ 没有子 Agent — 用 tmux 或自建
❌ 没有权限弹窗 — 用容器隔离
❌ 没有 Plan Mode — 用文件或自建
✅ 核心极简，一切可扩展
✅ 适配你的工作流，而非反过来
Pi 是一个终端 Coding Agent，核心设计理念是"极简 + 可扩展"。它不内置 MCP、子 Agent、权限弹窗、Plan Mode 等功能，而是把这些都交给扩展来实现。也就是说，Pi 只做好核心，剩下你来定。这正是它和其他编码 Agent 的最大区别——Pi 不会规定你的工作流。

// sessions

会话管理：树形结构 & 分支

Session 存储为 JSONL 文件，每条记录有 id 和 parentId，构成树形结构。可以在任意节点创建分支，无需新建文件。

/tree
原地导航会话树，跳到任意节点继续对话
/fork
从某个节点复制出新会话文件，继续探索
/compact
压缩旧消息，释放上下文窗口
session.jsonl
// 树形会话结构示意
[user msg] ─── [assistant] ─── [user msg]
                                ├── [assistant] ← 当前位置 (leaf)
                                └── [branch_summary]
                                     └── [user msg] ← 备选分支

// 会话命令
pi -c            # 继续最近的会话
pi -r            # 选择历史会话
pi --no-session  # 临时模式（不保存）

// 会话存储路径
~/.pi/agent/sessions/
  └── --Users-tanwei3-myproject/
       └── 20260526_a1b2c3d4.jsonl
Pi 的会话系统是用树形结构实现的，不是线性列表。每个条目有 id 和 parentId，这样你可以在对话的任何点创建分支，而不需要新建文件。/tree 命令让你在会话内跳转，/fork 把分支导出为新会话。/compact 用来压缩长对话，释放上下文窗口。

// compaction

上下文压缩：让长对话继续

当 token 数接近模型上下文窗口时，自动触发压缩——保留最近消息，旧消息摘要为结构化总结。

压缩流程

1. 从最新消息往前回溯，计算 keepRecentTokens
2. 找到截断点（用户/助手消息边界）
3. LLM 生成结构化摘要（Goal / Progress / Decisions）
4. 保存 CompactionEntry，保留 firstKeptEntryId
5. 后续对话使用 摘要 + 近期消息
settings.json
// ~/.pi/agent/settings.json
{
  "compaction": {
    "enabled": true,
    "reserveTokens": 16384,
    "keepRecentTokens": 20000
  }
}

压缩机制是 Pi 应对上下文窗口限制的核心策略。默认保留最近 20000 token 的消息，其余的由 LLM 生成结构化摘要。摘要格式包括 Goal、Progress、Key Decisions 等字段，还追踪了读写过的文件。压缩是自动触发的，也可以用 /compact 手动触发并追加自定义指令。

deep-dive · architecture

架构总览：Pi 的核心流程

用户输入 │ ▼ ┌─────────────────────────────────────────────────┐ │ Extension Commands → 匹配 /cmd? 直接执行 │ │ input 事件 → 可拦截/转换/处理 │ │ Skill/Template 展开 延迟加载指令 │ └─────────────────────────────────────────────────┘ │ ▼ before_agent_start → 可注入消息/修改 system prompt │ ▼ agent_start │ ▼ ┌──────── Agent Turn Loop ────────────┐ │ │ │ context → 可修改消息列表 │ │ before_provider_request │ │ LLM 调用 │ │ after_provider_response │ │ │ │ 工具调用? │ │ ├ tool_call → 可阻断/修改参数 │ │ ├ tool_result → 可修改结果 │ │ └ loop back │ └──────────────────────────────────────┘ │ ▼ agent_end
这是 Pi 的架构核心。用户输入进来后，先经过扩展命令检查，然后 input 事件可被拦截或转换。接着 skill 和 template 展开。然后 before_agent_start 可以注入消息或修改 system prompt。进入 Agent 循环后，每轮 LLM 调用前有 context 和 before_provider_request 钩子，LLM 调用后有 after_provider_response。如果 LLM 返回工具调用，tool_call 和 tool_result 事件都可以被拦截。整个流程中，扩展都可以在各环节介入。

deep-dive · extensions

扩展系统：Ti 的万能接口

my-extension.ts
import type { ExtensionAPI } from
  "@earendil-works/pi-coding-agent";
import { Type } from "typebox";

export default function (pi: ExtensionAPI) {
  // 监听事件
  pi.on("tool_call", async (event, ctx) => {
    if (event.toolName === "bash"
      && event.input.command?.includes("rm -rf")) {
      const ok = await ctx.ui.confirm(
        "危险!", "允许 rm -rf?");
      if (!ok) return { block: true };
    }
  });

  // 注册工具
  pi.registerTool({
    name: "deploy",
    label: "Deploy",
    description: "部署到生产环境",
    parameters: Type.Object({
      env: Type.String()
    }),
    async execute(id, params, sig, onUpd, ctx) {
      ctx.ui.notify(`Deploying to ${params.env}`);
      return { content: [{ type: "text",
        text: "Done!" }], details: {} };
    }
  });
}

扩展能做什么？

⚡ 注册自定义工具 (registerTool)
⚡ 注册命令 (registerCommand)
⚡ 注册快捷键 (registerShortcut)
⚡ 拦截/修改工具调用 (on tool_call)
⚡ 修改工具结果 (on tool_result)
⚡ 注入系统提示 (on before_agent_start)
⚡ 自定义压缩 (on session_before_compact)
⚡ 自定义 UI 组件 (ctx.ui.custom)

放置位置

全局: ~/.pi/agent/extensions/*.ts
项目: .pi/extensions/*.ts
包: pi install npm:@foo/pi-tools
扩展是 Pi 最强大的能力。一个 TypeScript 文件就能做到：注册自定义工具让 LLM 调用、拦截工具调用做权限控制、修改 LLM 请求/响应、注入系统提示、自定义压缩逻辑、甚至替换编辑器 UI。扩展通过 jiti 加载，原生支持 TypeScript 无需编译。

// skills

Skills：即插即用的能力包

Skill 是一个 Markdown 文件 + 辅助资源的目录，遵循 Agent Skills 标准，描述 + 指令分离，按需加载。

工作原理

1. 启动时扫描 Skill 目录，提取 name + description
2. 系统提示中包含可用的 Skill 列表（仅描述）
3. 任务匹配时，Agent 自动 read SKILL.md 加载完整指令
4. 也可以用 /skill:name 手动触发
brave-search/SKILL.md
---
name: brave-search
description: Web search and content extraction
  via Brave Search API. Use for searching
  documentation, facts, or any web content.
---

# Brave Search

## Setup
```bash
cd /path/to/brave-search && npm install
```

## Search
```bash
./search.js "query"              # 基础搜索
./search.js "query" --content    # 包含页面内容
```

## Extract Page Content
```bash
./content.js https://example.com
```

# Skill 目录结构
brave-search/
├── SKILL.md      # 必需：frontmatter + 指令
├── scripts/      # 辅助脚本
└── references/   # 参考文档（按需加载）
Skill 和 Extension 的区别在于：Skill 是声明式的 Markdown 指令，Agent 按需加载；Extension 是命令式的 TypeScript 代码，在启动时就加载。Skill 适合定义工作流、项目约定；Extension 适合添加新工具、拦截事件。两者配合使用，可以覆盖几乎所有需求。

// sdk

SDK：把 Agent 嵌入你的应用

sdk-quickstart.ts
import {
  createAgentSession,
  SessionManager,
  AuthStorage,
  ModelRegistry,
} from "@earendil-works/pi-coding-agent";

const authStorage = AuthStorage.create();
const modelRegistry = ModelRegistry.create(authStorage);

// 创建 Agent 会话
const { session } = await createAgentSession({
  sessionManager: SessionManager.inMemory(),
  authStorage,
  modelRegistry,
});

// 订阅事件流
session.subscribe((event) => {
  if (event.type === "message_update"
    && event.assistantMessageEvent.type === "text_delta") {
    process.stdout.write(
      event.assistantMessageEvent.delta);
  }
});

// 发送提示
await session.prompt("列出当前目录的文件");

四种运行模式

InteractiveMode — 完整 TUI
runPrintMode — 单次输出
runRpcMode — JSON-RPC 子进程集成
SDK — 嵌入 Node.js 应用

核心 API

session.prompt() 发送提示
session.steer() 转向消息
session.followUp() 追加消息
session.subscribe() 事件订阅
session.abort() 中止当前操作
SDK 让你把 Pi 的 Agent 能力嵌入任何 Node.js 应用。核心是 createAgentSession() 工厂函数，配合 SessionManager 管理。你可以订阅事件流实时获取 LLM 输出，可以在 Agent 工作时用 steer() 插入方向指令，用 followUp() 追加后续任务。也可以自定义工具、扩展、系统提示。SDK 是 Pi 走向集成的关键接口。

deep-dive · events

事件系统：扩展的钩子无处不在

会话生命周期
session_start · session_shutdown · session_before_switch · session_before_fork
Agent 流程
before_agent_start · agent_start · agent_end · turn_start · turn_end
消息流
message_start · message_update · message_end
工具执行
tool_call · tool_result · tool_execution_start/update/end
上下文 & Provider
context · before_provider_request · after_provider_response

常见用法

🛡️ tool_call → 危险命令拦截确认
📝 before_agent_start → 注入上下文/修改 prompt
🔄 context → 过滤/修改消息列表
💾 session_before_compact → 自定义压缩逻辑
📊 message_end → 计费/统计 hook

关键特性

✅ 事件处理是 异步 的（async/await）
✅ tool_call 的 event.input 是 可变的
✅ tool_result 可以 替换 返回内容
✅ ctx.ui.* 提供交互能力
✅ ctx.sessionManager 实时读取会话状态
事件系统是 Pi 扩展架构的核心。从会话生命周期到 Agent 执行流程，到消息和工具调用，每个环节都有钩子。最强大的两个：tool_call 事件可以拦截危险命令、修改参数；before_agent_start 事件可以注入系统提示或额外消息。这些钩子都是异步的，支持 async/await。

// session-format

会话格式：JSONL 树形结构

session.jsonl
// 第一行：会话头
{"type":"session","version":3,"id":"uuid",
 "cwd":"/path/to/project"}

// 用户消息
{"type":"message","id":"a1b2","parentId":null,
 "message":{"role":"user","content":"Hello"}}

// 助手回复
{"type":"message","id":"b2c3","parentId":"a1b2",
 "message":{"role":"assistant",
 "content":[{"type":"text","text":"Hi!"}],
 "provider":"anthropic","model":"claude-sonnet-4-5",
 "usage":{"input":50,"output":5}}}

// 工具调用结果
{"type":"message","id":"c3d4","parentId":"b2c3",
 "message":{"role":"toolResult","toolCallId":"call_1",
 "toolName":"bash","content":[{"type":"text",
 "text":"output..."}],"isError":false}}

// 压缩摘要
{"type":"compaction","id":"d4e5","parentId":"c3d4",
 "summary":"用户讨论了 X...","firstKeptEntryId":"c3d4",
 "tokensBefore":50000}

Entry 类型一览

session — 会话元数据（首行）
message — 对话消息（user/assistant/toolResult）
compaction — 上下文压缩摘要
branch_summary — 分支切换摘要
custom — 扩展持久化状态（不进 LLM）
custom_message — 扩展注入消息（进 LLM）
model_change — 模型切换记录
thinking_level_change — 思考级别变更
label — 用户书签标记
关键特性
✅ id + parentId → 天然树形结构
✅ 分支无需创建新文件
✅ SessionManager API 完整操作树
Pi 的会话存储用 JSONL 格式，每行一个 JSON 对象。最关键的设计是 id + parentId 构成的树形结构——你可以在任何节点创建分支，而不需要新建文件。Entry 类型涵盖消息、压缩、分支摘要、扩展状态等。SessionManager 提供了完整的 API 来遍历、分支、创建。

// takeaways

关键要点

1 · 极简核心

4 个内置工具 + 树形会话 + 上下文压缩，核心功能精简且完备。

2 · 事件驱动

从会话到 Agent 循环到工具调用，每个环节都有钩子，扩展无处不在。

3 · 三层扩展

Skill（声明式指令）、Extension（命令式代码）、Package（分发共享）。

4 · 树形分支

单文件树形结构，原地分支，/tree /fork /clone 灵活探索。

5 · SDK 可嵌入

createAgentSession() 一行创建 Agent，事件流实时订阅，可嵌入任何 Node 应用。

6 · 适配你的流程

Pi 不规定工作流——MCP、子 Agent、权限、Plan Mode 都通过扩展实现。
